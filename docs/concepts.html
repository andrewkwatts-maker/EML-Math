Core Concepts

The EML Sheffer Operator

A Sheffer operator is a single binary operation from which all other operations in a logical or mathematical system can be derived. For Boolean logic, NAND is the Sheffer stroke. For continuous real-valued mathematics, the EML operator plays the analogous role:

eml(x, y) = exp(x) − ln(y)

Odrzywolek (2026) proved that starting from eml and the constant 1, every elementary function can be expressed as a finite composition of eml calls. This includes addition, multiplication, trigonometric functions, logarithms, and transcendentals like π and e.

Key insight: eml(1, 1) = exp(1) − ln(1) = e. The mathematical constant e emerges naturally from the operator evaluated at its simplest input.

EMLPoint — Computation as State

An EMLPoint(x, y) is both a mathematical state and an expression tree node. Nesting EMLPoints is the computation — no separate AST class is needed:

from eml_math import EMLPoint
import math

# eml(x, y) = exp(x) - ln(y)
EMLPoint(1, 1).tension()          # e
EMLPoint(2, 1).tension()          # exp(2)

# ln(e) encoded as nested EMLPoints
# ln(x) = eml(1, eml(eml(1, x), 1)) 
EMLPoint(1, EMLPoint(EMLPoint(1, math.e), 1)).tension()  # 1.0

The Frame-Shift Guard (Axiom 8)

When y ≤ 0, ln(y) is undefined. The frame-shift guard automatically uses |y|, keeping all values in the real domain:

EMLPoint(1, -2).tension()   # uses |y| = 2  — always real

The Slipping Wheel Guard

When x > 709.78, exp(x) would overflow. The Slipping Wheel guard applies x ← ln(x), dampening the growth back into range:

EMLPoint(800, 1).tension()  # x=800 → x=ln(800) — no overflow

Mirror-Pulse Iteration

The Mirror-Pulse is the fundamental time-step of the EML system. One pulse advances the state from (x, y) to (|y|, T) where T = eml(x, y):

x_{t+1} = |y_t|
y_{t+1} = exp(x_t) − ln(|y_t|)

This creates a coupled oscillation between the growth coordinate x and the mirror coordinate y:

from eml_math import EMLState, EMLPoint, simulate_pulses, verify_conservation

s = EMLState(EMLPoint(1.0, 1.0))
traj = simulate_pulses(s, n_pulses=10)

# Axiom 10: T + ln(y) = exp(x) — always holds
print(verify_conservation(traj))   # True

for st in traj[:4]:
    print(f"n={st.flip_count}  rho={st.rho:.4f}")

EMLPair — Real Replacement for Complex Numbers

Complex numbers require two reals. The EML paper notes that sin, cos, and π require complex intermediates. EMLPair solves this using two EMLState objects in phase relationship — pure real arithmetic throughout:

from eml_math import EMLPair
import math

# Complex multiplication — no Python complex type used
z1 = EMLPair.from_values(3.0, 4.0)   # 3 + 4i
z2 = EMLPair.from_values(1.0, 2.0)   # 1 + 2i
z3 = z1 * z2                           # -5 + 10i
print(z1.modulus)                      # 5.0
print(z1.argument)                     # arctan(4/3)

# 4-frame rotation group: {×1, ×i, ×-1, ×-i}
frames = z1.frames()   # all 4 have identical Euclidean delta

t' = t·cosh(φ) − (s/c)·sinh(φ)

The 3:1 Flip Cycle (Axiom 9)

Each complete cycle consists of 3 growth pulses followed by 1 mirrored reflection. This asymmetry yields a net surplus of FLIP_YIELD = 2 reality units per cycle — a discrete analog of the matter-antimatter asymmetry:

from eml_math import EMLState, EMLPoint

s = EMLState(EMLPoint(1.0, 1.0))
s4 = s.three_one_flip()   # 4 pulses: 3 growth + 1 reflection
print(s4.tread_yield())   # net reality units accumulated

Discrete Mode (Planck Scale)

Setting D enables discrete quantisation via round(T × D). At D = PLANCK_D ≈ 6.187 × 10³⁴, the quantization step corresponds to the Planck length:

from eml_math.constants import PLANCK_D
from eml_math import EMLPoint

p = EMLPoint(1.0, 1.0, D=PLANCK_D)   # Planck-scale discrete mode
p2 = p.mirror_pulse()                   # y_new = round(T × D) / D
